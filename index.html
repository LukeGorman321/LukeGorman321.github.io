Mad Lad in a Mad Lab

Fire Spread

The feature I implemented which is displayed here is the fire's ability to spread faster onto wooden objects.

This allows the player to control the spread of the fire by moving wooden crates and barrels where they want the fire to spread

One difficulty in creating this was that due to the random nature of the fire's spread, it was not always clear if an adjustment to the code helped make the fire spread in the intended manner. So, I put the settings very high to make the fire spread comically onto wooden things, that way I could clearly see and easily test the algorithm.

Minecraft Minigames using Datapacks

Candy Cane Catapult

One minigame I made in Minecraft had a very large map, which was intended to make the game slightly slow, forcing players to manage their resources carefully.

However, this created a problem, since the game ended up going much slower than intended. So, I moved bases closer together, but this still wasn't enough

So, I added items that help the player go faster, but it was still too slow. Finally, I created a slightly more fun solution, which was a catapult which could launch the player to the center of the map instantly, making it much faster to get around.

Networking

One game I created involves people on several devices connecting over the internet.

Since I had never done something like this before, I first created a simple counter where whenever one person clicks the increment button, the number goes up on both screens

There were several challenges with this, but by doing this simple task, I learned how to navigate networking for the rest of the project.

For example, one challenge was that the information was all stored on the host's side, so whenever the person who wasn't the host tried to increment the number, it would just get overridden by what the host already had.

So, rather than just storing a single variable where the total increment is stored, I also stored information to say when a particular person was pressing the button, which was fed in by each player. Then, the host would see this, and increment the count accordingly.
